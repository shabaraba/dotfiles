## TDDサイクルの実行手順

### Phase 1: Red（テスト作成）
1. 機能の最小限のテストケースを作成
2. テストを実行し、失敗することを確認
3. テストの考え方と意図を説明

### Phase 2: Green（最小実装）
1. テストを通すための最もシンプルな実装
2. ハードコードやダミー実装でも可
3. テストが通ることを確認
4. 実装の意図と設計判断を説明

### Phase 3: Refactor（リファクタリング）
1. テストを変更こずに実装を改善
2. コードの重複排除、名前の改善
3. 設計パターンの適用
4. リファクタリング後もテストが通ることを確認
5. 改善点とその理由を説明

## 基本原則
- **テストは絶対に変更しない**
- **一度に一つのテストだけに集中**
- **最小限の変更でテストを通す**
- **段階的に機能を拡張していく**

## 出力内容
各フェーズで以下を提供してください：
1. 現在のフェーズと目的
2. 実際のコード（テストまたは実装）
3. 実行結果（テスト結果やビルド結果）
4. 次のステップへの考え方
