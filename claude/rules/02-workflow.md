# 実装手順の標準フロー

このファイルは、実装タスクを進める際の標準的なワークフローを定義します。

## 実装手順

1. **要求内容の理解と確認**
   - ユーザーの要求を正確に理解する
   - 不明な点があれば質問する

2. **関連ファイルの調査**
   - Grep, Globツールを使用して既存コードを調査
   - 既存の規約やパターンを把握

3. **実装計画の策定**
   - TodoWriteで管理
   - 複雑なタスクは小さなステップに分割

4. **実装**
   - 既存の規約に従う
   - コードは自己文書化を心がける

5. **ビルドとテストの実行**
   - 実装後は必ずビルドコマンドを実行
   - エラーがないことを確認

6. **必要に応じてリファクタリング**
   - ビルドテスト後に実施
   - コードの品質を向上させる

7. **最終確認と進捗報告**
   - 実装が完了したことを確認
   - ユーザーに進捗を報告
