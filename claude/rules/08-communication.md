# コミュニケーション方針

このファイルは、ユーザーとのコミュニケーションに関する方針を定義します。

## 進捗報告

### タイミング
- 主要なステップごとに進捗を報告
- 長時間かかる作業の場合は途中経過も報告
- ユーザーが現在の状況を把握できるようにする

### 報告内容
- 何を完了したか
- 次に何をするか
- 問題が発生した場合はその内容

## 質問のタイミング

### 即座に質問すべき場合
- 不明な点や曖昧な要求がある場合
- 設計上の判断が必要な場合
- 複数の実装方法があり、どれを選ぶか迷う場合

### 提案の仕方
- 実装の選択肢がある場合は、理由を説明した上で提案
- 推奨する方法とその理由を明確にする
- 代替案も提示することで、ユーザーが判断しやすくする

## 困ったときの対応

### エラー解決に苦戦した場合
- 遠慮なくユーザーに相談する
- 試したことと現状を報告
- どこで詰まっているかを明確に伝える

### 3回試行ルール
- ビルドやテストの失敗を3回修正してもまだ解決できない場合
- 現状報告と困っている点をユーザーに伝える
- アドバイスを求める
- 無理に解決しようとせず、相談する

## 言語設定

- **応答言語**: 日本語で応答する
- **コード内コメント**: 必要最小限
- **コミットメッセージ**: 英語（Semantic Commit Messages）
- **PRタイトル**: 英語（Semantic Commit Messages）
- **PR本文**: 日本語
